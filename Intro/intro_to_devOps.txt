What is DevOps?
DevOps is a combination of Development (Dev) and Operations (Ops).
It is a culture, practice, and set of tools that increases an
organizationâ€™s ability to deliver applications and services at a
high speed with improved quality, reliability, and collaboration.

------------------------------------------------------------

Dev vs Ops:
Developer:
    - Writes the code
    - Fixes bugs
    - Adds new features

Operations:
    - Hosts the application
    - Deploys new releases
    - Monitors system health
    - Maintains servers and infrastructure
    - Scales the application

DevOps bridges the gap between both teams.

------------------------------------------------------------

DevOps Objectives:
- Maintain the Code
- Deploy the Code
- Monitor the Code
- Test the Code
- Automate the Pipeline
- Scale the Application
- Improve Collaboration between Dev and Ops

------------------------------------------------------------

DevOps in the Software Development Life Cycle:
1. Plan
2. Develop
3. Build
4. Test
5. Release
6. Deploy
7. Operate
8. Monitor
(Feedback goes back to planning)

------------------------------------------------------------

Continuous Integration (CI):
- Developers push code frequently
- Automated build & test triggers
- Ensures code reliability and quality

Continuous Deployment/Delivery (CD):
- Automatically deploys the tested code
- Fast and safe releases
- Reduces manual errors
